HTTP 重新導向至 HTTPS

Created2024-03-28|Updated2024-05-18|DevelopmentInternet
|Post View:

HTTPS 的優勢

  1. 安全性：將 client 與 server 間傳遞的訊息加密，HTTPS (Hyper Text Transfer Protocol Secure) 字面意思即為更加安全的 HTTP。
  2. 提升 SEO 排名：Google 官方 已將 HTTPS 作為網站搜尋排名影響的指標之一。

強制網站執行 HTTPS 的方法

加入 HSTS Header

HSTS (HTTP Strict Transport Security): 在 response header 中加入 Strict-Transport-Security: max-age=31536000; includeSubDomains 來要求瀏覽器將 HTTP 轉為 HTTPS，但初次訪問網站時，仍有可能因為還未收到 HSTS 而使用 HTTP 連線。

301 redirect

可以透過設定 web server 來將所有 HTTP 流量轉向 HTTPS

status codes

status codes 由三位整數組成，介於 100 ~ 599，用來表示 request 的結果。

其中第一位數字定義了 response 的類別：

  • 1xx (Informational)
  • 2xx (Successful)
  • 3xx (Redirection)
  • 4xx (Client Error)
  • 5xx (Server Error)

301 即代表我們要存取的資源已經永久 (permanently) 被移到另一個 URI，瀏覽器可以根據 response header 中的 Location 來做重新導向。
但要注意的是 301 同時也會將 POST 轉為 GET，若要保留 request method 可以使用 308。
